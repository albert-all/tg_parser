# Telegram Bot Mode

Этот документ описывает запуск и использование бота `tg_bot.py`, который работает от имени пользователей через их Telegram-аккаунты (Telethon + QR auth), хранит пользовательские темы в PostgreSQL и выдает результаты в чат + CSV.

## 1. Что умеет бот

- QR-авторизация каждого пользователя отдельно.
- 2FA-пароль (если включен у пользователя).
- Персональные темы:
  - создать/удалить тему;
  - добавить/удалить чат;
  - добавить/удалить ключевое слово.
- Поиск по теме с фильтром дат.
- Автопроверка темы по расписанию (в минутах) с уведомлением при новых совпадениях.
- Отправка результатов:
  - текстом в чат;
  - CSV-файлом;
  - или оба варианта.
- Отмена активного поиска.
- Ретеншен 30 дней для истории запусков и сохраненных CSV/сессий.

## 2. Подготовка переменных окружения

1. Скопируйте `.env.bot.example` в `.env` (или используйте существующий `.env`, добавив нужные поля):

```bash
cp .env.bot.example .env
```

2. Заполните:

- `BOT_TOKEN` - токен Telegram-бота.
- `TG_API_ID`, `TG_API_HASH` - Telethon API credentials.
- `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`.

## 3. Запуск в Docker (рекомендуется, 24/7)

Из папки `tg_parser`:

```bash
docker compose up -d --build
```

Логи:

```bash
docker compose logs -f bot
```

Остановка:

```bash
docker compose down
```

## 4. Локальный запуск без Docker

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements_bot.txt
python tg_bot.py
```

Также нужен доступный PostgreSQL и корректный `DATABASE_URL`.

## 5. Пользовательский сценарий в боте

1. Нажать `/start`.
2. Нажать `Авторизация` (или `/auth`), отсканировать QR.
3. Нажать `Проверить QR` (или `/auth_check`).
4. Если бот просит 2FA - отправить пароль следующим сообщением.
5. Создать тему:
   - `/theme_new news`
6. Добавить чаты:
   - `/theme_add_chat news @rian_ru`
   - `/theme_add_chat news -1001234567890`
   - `/theme_add_chat news https://t.me/some_channel`
   - `/theme_add_chat news Радиоактивные люди`
7. Добавить ключевые слова:
   - `/theme_add_kw news Москва`
   - `/theme_add_kw news погодa`
8. Запустить поиск:
   - `/search news --date-from 2026-02-01 --date-to 2026-02-27 --limit 50 --format both`
9. При необходимости остановить:
   - `/cancel`
10. Включить периодическую проверку темы:
   - `/watch_set news 60`
   - или кнопками: `Подписки` -> `Включить` -> выбрать тему -> выбрать интервал
   - для произвольного значения: кнопка `Свой период` -> отправьте число минут (например `45`)
11. Посмотреть активные подписки:
   - `/watch_list`
12. Отключить проверку:
   - `/watch_off news`
   - или кнопками: `Подписки` -> `Отключить`

## 6. Команды

- `/start`
- `/help`
- `/auth`
- `/auth_check`
- `/auth_refresh`
- `/logout`
- `/themes`
- `/theme_new <name>`
- `/theme_delete <name>`
- `/theme_add_chat <theme> <chat_ref>`
- `/theme_del_chat <theme> <chat_ref>`
- `/theme_add_kw <theme> <keyword>`
- `/theme_del_kw <theme> <keyword>`
- `/watch_set [theme] <minutes>`
- `/watch_off [theme]`
- `/watch_list`
- `/search <theme> [--date-from YYYY-MM-DD] [--date-to YYYY-MM-DD] [--limit N] [--format both|text|csv]`
- `/cancel`

## 7. Важные замечания

- Поиск идет от имени аккаунта пользователя, который авторизовался через QR.
- Ссылки на сообщения отправляются всегда, когда Telegram позволяет их сформировать.
- Для некоторых приватных диалогов прямой URL может отсутствовать.
- Если для пользователя уже идет поиск, новый запуск блокируется до `/cancel` или завершения текущего.
- Автопроверка ищет новые сообщения только с момента предыдущего прохода, чтобы не дублировать старые результаты.
- В кнопочных разделах (`Главное меню`, `Темы`, `Поиск`, `Подписки`, `Авторизация`) бот переиспользует/обновляет одно сообщение интерфейса, чтобы не засорять чат.
- Также бот закрепляет нижнюю клавиатуру быстрых действий (`Главное меню`, `Темы`, `Поиск`, `Подписки`, `Помощь`), чтобы кнопки не «уезжали» вверх в истории.
